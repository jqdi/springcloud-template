# Agent Skills 使用指南

## 概述
本项目使用Agent Skills系统来记录和管理项目的规范、最佳实践和改动历史，帮助开发者快速理解和维护项目。

## 目录结构
```
skills/
├── standards/           # 项目规范文档
│   ├── project-standard.md     # 项目整体规范
│   └── ...
├── modules/             # 模块规范文档
│   ├── module-standard.md      # 模块开发规范
│   └── ...
├── changes/             # 改动记录
│   ├── change-template.md      # 改动记录模板
│   └── 2024-01-changes.md      # 2024年1月改动记录
│   └── ...
└── README.md           # 本使用指南
```

## 使用方法

### 1. 查找项目规范
当需要了解项目规范时，可以查阅以下文档：
- `skills/standards/project-standard.md` - 项目整体架构和规范
- `skills/modules/module-standard.md` - 各模块开发规范

### 2. 记录改动
每次对项目进行重要改动时，请按以下步骤操作：
1. 参考 `skills/changes/change-template.md` 中的模板
2. 在 `skills/changes/` 目录下创建新的改动记录文件
3. 按月份命名文件，如 `2024-01-changes.md`
4. 按模板要求填写改动详情

### 3. 遵循开发规范
- 在开发新功能前，先阅读相关模块的规范文档
- 确保代码风格与项目规范一致
- 使用统一的异常处理和日志记录方式

## 规范分类

### 代码规范
- 命名规范：类名、方法名、变量名的命名约定
- 包结构：包的组织方式和职责划分
- 注释规范：代码注释的标准和要求
- 代码质量：代码复杂度、重复率等指标要求

### 架构规范
- 分层架构：Controller、Service、DAO等层次职责
- 模块划分：各模块的职责和依赖关系
- 接口设计：API设计原则和规范
- 数据库设计：表结构设计和规范

### 部署规范
- 环境配置：不同环境的配置差异
- 发布流程：代码提交、构建、部署的流程
- 监控告警：系统监控和告警配置
- 日志管理：日志格式和管理规范

## 维护规则

### 更新频率
- 项目整体规范：重大架构调整时更新
- 模块规范：模块重构或职责变化时更新
- 改动记录：每次重要改动都要记录

### 维护责任
- 项目负责人：负责整体规范的制定和维护
- 模块负责人：负责各自模块规范的维护
- 所有开发者：负责记录自己所做的重要改动

### 审查机制
- 代码审查时检查是否符合规范
- 定期回顾规范的有效性和完整性
- 根据项目发展不断完善规范

## 注意事项
1. 所有规范文档应保持最新状态
2. 重要改动必须记录，确保可追溯
3. 新加入团队的成员应首先阅读相关规范
4. 规范不是一成不变的，可根据实际需要进行调整
